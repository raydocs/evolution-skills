# Skills 健康报告

显示所有保存的 skills 的使用统计和健康状态。

## 工作流

### Step 1: 扫描所有 skill 文件

```bash
ls ~/.factory/skills/*.md 2>/dev/null
```

### Step 2: 提取反馈标记

从每个文件中提取 `<!-- feedback: ... -->` 标记:

```
count=N | success=N | failed=N | last_used=DATE
```

### Step 3: 生成报告

```markdown
## Skills 健康报告

生成时间: {当前日期}

### 按技术栈

| 技术栈 | 模式数 | 平均成功率 | 状态 |
|-------|-------|-----------|------|
| flutter | 5 | 85% | ✅ |
| swiftui | 3 | 60% | ⚠️ |

### 模式详情

| 模式 | 使用 | 成功 | 失败 | 成功率 | 状态 |
|------|-----|------|------|-------|------|
| Flutter 下拉刷新 | 10 | 9 | 1 | 90% | ✅ 健康 |
| SwiftUI 导航 | 5 | 3 | 2 | 60% | ⚠️ 需审查 |

### 需要关注

- ❌ `react-hooks-cleanup`: 0% 成功率，建议修复或移除
- ⚠️ `swiftui-navigation`: 60% 成功率，需要审查边缘情况

### 建议

1. 修复失败率 >50% 的模式
2. 审查 3+ 次失败的模式
3. 考虑移除长期未使用的模式
```

### Step 4: 提供操作选项

```
你可以:
1. `/evolve {tech} {pattern}` - 更新特定模式
2. 直接编辑 ~/.factory/skills/{file}.md
3. 删除不再需要的模式
```

## 健康状态定义

| 成功率 | 状态 | 建议 |
|-------|------|------|
| > 90% | ✅ 健康 | 无需动作 |
| 70-90% | 🔶 良好 | 可选审查 |
| 50-70% | ⚠️ 需审查 | 检查边缘情况 |
| < 50% | ❌ 问题 | 需要修复或移除 |

## 手动验证

如果需要完整验证所有 skills:

```bash
# 列出所有 skill 文件
find ~/.factory/skills -name "*.md" -exec echo "=== {} ===" \; -exec head -20 {} \;
```
